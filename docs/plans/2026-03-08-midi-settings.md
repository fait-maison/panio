# MIDI Settings Redesign — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Move MIDI device selection out of the navbar dropdown and into the settings page as a proper Select, making the navbar badge read-only.

**Architecture:** `MidiStatus.svelte` becomes a plain `<a href="/settings">` badge (dot + label). The settings page gains a MIDI section using shadcn Select bound to `midi.inputList` and `midi.setPreferredDevice()`. No store changes needed.

**Tech Stack:** SvelteKit 2, Svelte 5 runes, shadcn-svelte 1.x Select (bits-ui 2.x), Playwright e2e tests.

### Task 4: Add MIDI section to settings page

**Files:**

- Modify: `src/routes/settings/+page.svelte`

**Step 1: Look up the Select API**

Before writing code, check the shadcn-svelte Select docs via context7 to confirm the exact prop names (`value`, `onValueChange`, `disabled`, `Select.Item value`, etc.) for the installed version.

**Step 2: Update the script block**

Replace the existing imports block. Add `midi` store and Select imports, remove `MidiStatus`:

```svelte
<script lang="ts">
	import * as ToggleGroup from '$lib/components/ui/toggle-group/index.js';
	import * as Select from '$lib/components/ui/select/index.js';
	import { settings } from '$lib/stores/settings.svelte';
	import type { KeyboardSize, ProgressionNotation } from '$lib/stores/settings.svelte';
	import { t, locale, type Locale } from '$lib/i18n.svelte';
	import { midi } from '$lib/stores/midi.svelte';

	const KEYBOARD_SIZES: { value: KeyboardSize; label: string }[] = [
		{ value: 's', label: 'S' },
		{ value: 'm', label: 'M' },
		{ value: 'l', label: 'L' }
	];
</script>
```

**Step 3: Replace the MIDI section in the template**

Find this block (around line 87-92):

```svelte
<hr class="divider" />

<section class="midi-section">
	<MidiStatus />
</section>
```

Replace with:

```svelte
		<hr class="divider" />

		<section>
			<h2>MIDI</h2>
			<Select.Root
				type="single"
				value={midi.preferredDeviceId ?? ''}
				onValueChange={(v: string) => midi.setPreferredDevice(v || null)}
				disabled={midi.inputList.length === 0}
			>
				<Select.Trigger class="w-full">
					<Select.Value
						placeholder={midi.inputList.length === 0
							? t('midi.noDevices')
							: t('midi.noDevice')}
					/>
				</Select.Trigger>
				<Select.Content>
					{#if midi.preferredDeviceId}
						<Select.Item value="">{t('midi.noDevice')}</Select.Item>
					{/if}
					{#each midi.inputList as device (device.id)}
						<Select.Item value={device.id}>{device.name}</Select.Item>
					{/each}
				</Select.Content>
			</Select.Root>
		</section>
```

**Step 4: Clean up CSS**

Remove the `.midi-section` rule from the `<style>` block:
